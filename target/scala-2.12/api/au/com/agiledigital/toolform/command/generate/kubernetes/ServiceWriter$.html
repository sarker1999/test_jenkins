  28.  def writeService(service: ToolFormService): Result[Unit]

    Writes a Kubernetes "service" specification based on the provided toolform service.

    Writes a Kubernetes "service" specification based on the provided toolform service.

    A service is an abstraction to provide a stable interface to one or more pods. It specifies the DNS the name used to access a deployment.

    For example, an update could be deployed which means a pod running an old version of a server gets destroyed and a new pod with the new version of the server is created. Although the deployment has changed, the service remains static and dependent services don't need to be given a new address to access the new pod.

    service

    the service object that will be used to create the deployment spec.

    returns

    a state monad encapsulating the context of the writing process after the method has completed.

    See also

    https://kubernetes.io/docs/concepts/services-networking/service/
